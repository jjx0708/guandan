你需要帮我开发一个网页版掼蛋游戏。请按照以下要求开发：

## 项目要求

1. **技术栈**：纯前端 HTML + CSS + JavaScript，使用 PeerJS (WebRTC) 实现点对点连接
2. **无需服务器**：所有逻辑在客户端运行，房主手机作为主机
3. **手机端友好**：响应式设计，支持手机浏览器

## 游戏规则（必须完整实现）

### 牌型（从大到小）
1. 大王（红桃大小王是大王，其他花色是小王）
2. 小王
3. 级牌（如打A时，A是级牌；打2时，2是级牌）
4. 同花顺（花色相同的顺子）
5. 炸弹（4张及以上相同牌）
6. 钢板（三张相同+三张相同）
7. 顺子（5张及以上连续单牌）
8. 对子（2张相同）
9. 单张

### 贡牌规则
- 头游给四游贡最大的牌，四游还牌（≤10）
- 二游给三游贡最大的牌，三游还牌（≤10）
- 如果上游双下，则三游和四游都需要贡牌
- 供牌必须供手里最大的牌：如大小王、级牌（两张级牌有一张逢人配可以供另一张，保留红心级牌），依次类推
- 还牌只能还小于等于10以下的牌

### 记分规则（三一二制）
- 头游：+3分
- 二游：+1分
- 三游：-1分
- 四游：-3分

### 升级规则
- 头游+二游（搭档）升级
- 如果双下（头游+二游是搭档），直接升3级

### 逢人配（万能牌）
- 每局随机生成一张逢人配，可以代替任何牌
- 逢人配不能单独出，只能作为配牌使用
- 级牌中的红心级牌（如打A时红桃A）是固定的逢人配

## 房主功能
1. 创建房间，生成房间号
2. 踢人功能
3. 开始游戏
4. 重开功能

## UI 设计要求
1. 简洁美观，适合手机操作
2. 显示：手牌、出牌区、队友出牌、对手出牌
3. 显示当前级别、已方得分、累计记分
4. 出牌时有明显的牌型提示
5. 贡牌/还牌时有专门的交互界面

## 文件结构
请创建以下文件：
- index.html      # 主页面
- css/style.css   # 样式
- js/game.js      # 游戏逻辑
- js/peer.js      # PeerJS 连接管理
- js/cards.js     # 牌型判断和比较逻辑

开发完成后请汇报进度。
